노출 지면이 늘어난다고, 전체 수익도 늘어날까요?
카카오페이는 기존 오퍼월로 안정적 수익을 내고 있는 매체였습니다.
새 지면을 추가하면 기존 오퍼월 수익은 줄어들 것이라는 우려도 있었지만
기존 오퍼월을 유지한 채로, 혜택메인 · 만보기 · 출석체크 3개 지면에
API 방식으로 추가 지면을 테스트했습니다.
그 결과, 기존의 오퍼월 수익도 성장했으며, 새로운 수익원이 추가되었습니다.
도입 흐름
10월 기존 오퍼월 운영 중 · 추가 지면 도입 논의 시작
11월 API 연동으로 3개 지면 확대 — 혜택메인, 만보기, 출석체크
12월 추가 지면의 수익이 전체 수익의 67.8%로 성장 · 오퍼월 수익도 동반 상승

두 채널이 경쟁하는 것이 아니라 서로 상승 작용을 일으킨 것이 핵심이었습니다.

1

2
브랜드 커스터마이징
재화 아이콘, 텍스트 색상, 로딩 뷰를 앱 디자인에 맞게 덮어씌웁니다. 코드 추가 없이 renderer 설정만으로 완성.
3
완전 커스텀 레이아웃 (선택)
MCNativeAdRenderer 인터페이스로 자유로운 카드 UI 구현. SDK는 데이터·트래킹만 담당, UI 전권은 파트너사에.
4
데이터 활용
MCCampaign 객체로 캠페인명·리워드·프로모·진행상태 등 실시간 데이터 접근. 진행 중 배지·프로모 배지 자동 표시.

SDK는 이미 연동되어 있으니까요
A
직접 시작하기 — SDK 업데이트만 하면 됩니다
이미 SDK가 연동된 상태입니다. Native SDK 버전을 최신으로 올리고, MCNativeAdView를 원하는 지면에 드롭인하면 바로 캠페인이 노출됩니다.
B
MAF 팀과 함께 시작하기 — 애매하면 도와드립니다
어느 지면에 넣을지, 어떤 포맷이 맞을지 고민되신다면 MAF 팀이 함께 검토해 드립니다. 연락 한 번으로 시작할 수 있습니다.
